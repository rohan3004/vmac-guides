macOS on VMware

A step-by-step guide for beginners to install a fully-functional macOS virtual machine on a Windows PC.

Last Updated: October 14, 2025 | Author: Rohan Chakravarty

Welcome! This guide is designed to be a clear and comprehensive walkthrough for anyone new to virtualizing macOS. We will go through every step in detail, explaining not just what to do, but why you're doing it. By the end, you'll have a working macOS virtual machine for development, testing, or exploration.

Phase 1: Gathering Your Tools (Requirements)

Before we begin, you need to download all the necessary software. It's best to create a dedicated folder (e.g., C:\VMware-macOS) and save everything there to stay organized.

1. Python

Why you need it: We need a script to download the official macOS files from Apple, and that script is written in Python.
Action: Go to the official Python website and download the latest Windows installer. During installation, make sure to check the box that says "Add Python to PATH".

Download Python

2. OpenCorePkg

Why you need it: Think of this as a special key that tricks the macOS installer into thinking it's on a real Mac. It contains the Python script we need.
Action: Download the latest RELEASE version (it will be a ZIP file).

Download OpenCorePkg

3. QEMU for Windows

Why you need it: Apple's installer file (.dmg) is not compatible with VMware. QEMU contains a tool that can convert it to the correct format (.vmdk).
Action: Download the most recent 64-bit installer and install it.

Download QEMU

4. VMware Workstation Pro

Why you need it: This is the main program (the "hypervisor") that will create and run our virtual machine.
Action: You must search for, download, and install this application. Make sure it is installed before proceeding.

5. VMware Unlocker

Why you need it: By default, VMware for Windows doesn't let you install macOS. This patch "unlocks" that feature.
Action: Download the latest version (a ZIP file).

Download Unlocker

Phase 2: Preparing the macOS Installer

In this phase, we'll use the tools we downloaded to create a bootable installer file for VMware.

1

Download the macOS Recovery Image

First, extract the OpenCorePkg-....zip file. Inside, navigate to the Utilities folder, and then into the macrecovery folder. In the address bar of File Explorer, type cmd and press Enter. This will open a command prompt in the correct location.

Now, run the Python script to download the files. This command downloads macOS Monterey, which is a stable choice.

python macrecovery.py -b Mac-E43C1C25D4880AD6 -m 00000000000000000 download
2

Convert the Image to VMDK

After the download, you will see a new folder named com.apple.recovery.boot. Inside is a file called BaseSystem.dmg. We need to convert this. Open a new command prompt or PowerShell window and run the following command. Note that if you installed QEMU elsewhere, you'll need to adjust the path.

"C:\Program Files\qemu\qemu-img.exe" convert -O vmdk "C:\path\to\your\folder\com.apple.recovery.boot\BaseSystem.dmg" "C:\path\to\your\folder\BaseSystem.vmdk"

Easy Path Tip

Instead of typing the long paths, you can drag and drop the qemu-img.exe and BaseSystem.dmg files directly into the command prompt window. Just make sure to add the other parts of the command correctly.
3

Patch VMware with the Unlocker

This step enables the "Apple Mac OS X" option in VMware.

  1. Important: Close VMware Workstation completely. Go to Task Manager (Ctrl+Shift+Esc) and ensure no VMware processes are running.
  2. Extract the Unlocker zip file.
  3. Right-click on the win-install.cmd script and choose "Run as administrator". A black window will appear and run some commands. Wait for it to finish and close automatically.

Phase 3: Creating and Configuring the VM

Now we will build the virtual machine inside VMware.

4

Create the Virtual Machine

Open VMware Workstation and follow the wizard:

  1. File > New Virtual Machine > Select "Typical (recommended)".
  2. Select "I will install the operating system later".
  3. Select "Apple Mac OS X" and choose the latest version (e.g., macOS 12). If this is not visible, the Unlocker patch failed.
  4. Give your VM a name (e.g., "macOS Monterey") and choose where to save it.
  5. Set the disk size to at least 80 GB (128 GB is better). Select "Store virtual disk as a single file".
  6. On the final screen, click "Customize Hardware".
5

Configure Virtual Hardware

In the hardware customization window:

  1. Memory: Give it at least 8 GB (8192 MB).
  2. Processors: Give it at least 4 cores.
  3. Hard Disk (SCSI/SATA): Select this and click the "Remove" button at the bottom. We will add our own.
  4. Click "Add...", choose "Hard Disk", select "SATA", then "Use an existing virtual disk". Browse and select the BaseSystem.vmdk file you created earlier.
  5. CD/DVD Drive: Select this and on the right side, uncheck "Connect at power on".
  6. Click "Close" and then "Finish".
6

Edit the .vmx Configuration File

This is the most important step. Find where you saved your VM. There will be a file with a .vmx extension (e.g., macOS Monterey.vmx). Right-click it and open with Notepad.

Scroll to the very bottom and paste these lines exactly as they are:

smc.version = "0"
ethernet0.virtualDev = "vmxnet3"
board-id.reflectHost = "FALSE"
board-id = "Mac-937A206F2EE63C01"
hw.model.reflectHost = "FALSE"
hw.model = "MacBookPro15,1"
serialNumber.reflectHost = "FALSE"
serialNumber = "C01234567890"
smbios.reflectHost = "TRUE"

AMD CPU Users

If your computer has an AMD processor, you MUST add more lines. Paste the entire block of `cpuid` lines from the original document into your `.vmx` file as well.

Save and close the file.

Phase 4: Installing macOS

The preparation is complete. It's time to install the operating system.

7

Boot and Format the Drive

In VMware, select your new VM and click "Power on". After a few moments, you should see the macOS Recovery screen.

  1. Select "Disk Utility" and continue.
  2. In the top menu bar of Disk Utility, click View > Show All Devices.
  3. On the left, select the top-level entry for your main disk (e.g., "VMware Virtual SATA Hard Drive Media").
  4. Click the "Erase" button. Name it Macintosh HD, set the Format to APFS, and Scheme to GUID Partition Map. Click Erase.
  5. When finished, close Disk Utility by clicking the red button.
8

Install the Operating System

You are now back at the main recovery screen.

  1. Select "Reinstall macOS" and click Continue.
  2. Follow the on-screen prompts, agreeing to the license terms.
  3. When asked where to install, select the "Macintosh HD" drive you just created.

The installation will now begin. It will download the full OS from Apple and may take a long time (30 minutes to over an hour) and restart several times. This is normal.

Phase 5: Post-Installation Cleanup

Once the installation is complete, you'll be greeted with the macOS setup assistant. Complete this as you would on a new Mac.

9

Install VMware Tools

This is a critical step for good performance.

  1. With the VM running, go to the VMware top menu: VM > Install VMware Tools...
  2. A virtual disc will appear on the macOS desktop. Double-click it and run the installer.
  3. You will need to go to System Preferences > Security & Privacy and click "Allow" to enable the system extension. Follow the prompts and restart the VM when finished.
10

Final Cleanup

Shut down the macOS VM. Go back into the VM settings. Select the "Hard Disk (SATA)" that points to your BaseSystem.vmdk file and click "Remove". Your VM will now boot from the main drive you installed macOS onto.

Troubleshooting

Stuck on the Apple Logo?

This is the most common problem. It is almost always a typo in your .vmx file. Open it again and carefully check every single character of the lines you added.
