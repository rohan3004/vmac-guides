macOS on VMware

The definitive, expert-level guide to installing a fully-functional macOS virtual machine on VMware Workstation Pro using the modern OpenCore bootloader method.

Last Updated: October 14, 2025 | Based on guide by Rohan Chakravarty

Virtualizing macOS offers a powerful, sandboxed environment for developers, security researchers, and enthusiasts. This guide provides a comprehensive walkthrough, expanded with technical explanations to not only ensure a successful installation but to help you understand the mechanics behind it.

Prerequisites & Tools Deep Dive

Gathering the correct tools is the foundation of a successful installation. Here’s a detailed look at each component's role.

1. OpenCorePkg (Latest Release)

What it is: OpenCore is not a patch, but a sophisticated bootloader. Its job is to perform a series of "in-memory" modifications before the macOS kernel is loaded.

Download OpenCorePkg

2. QEMU for Windows

What it is: We only need its qemu-img.exe utility, a powerful tool for converting Apple's .dmg recovery image into the .vmdk format that VMware uses.

Download QEMU for Windows

3. VMware Unlocker

What it is: A community-made patch that modifies VMware's program files to lift the restriction on running macOS and add the necessary guest OS profiles.

Download Unlocker

4. Python 3.x

What it is: Required to run the macrecovery.py script, which connects to Apple's servers to download the official macOS recovery image.

Download Python

The Installation Process

1

Acquire & Convert Recovery Image

Use the macrecovery.py script to download the BaseSystem.dmg. Then, convert it to a VMware-compatible .vmdk using the QEMU utility.

& "C:\Program Files\qemu\qemu-img.exe" convert -O vmdk -o compat6 BaseSystem.dmg BaseSystem.vmdk
2

Patch VMware & Create VM

Run the Unlocker's win-install.cmd as an Administrator. Afterward, create a new "Apple Mac OS X" virtual machine with 8GB+ RAM, 4+ CPU cores, and a 128GB+ disk.

3

Modify the .vmx File

This is the most critical step. Add the following parameters to the end of your .vmx file to make the VM identify itself as a real Mac.

smc.version = "0"
board-id = "Mac-E43C1C25D4880AD6"
hw.model = "MacPro7,1"
serialNumber = "F5K99999P7QM"
ethernet0.virtualDev = "vmxnet3"
4

Install & Finalize macOS

Boot the VM into macOS Recovery, erase your main drive as APFS in Disk Utility, and reinstall macOS. After setup, install VMware Tools, then shut down and remove the BaseSystem.vmdk from the VM settings.

Post-Install Optimizations

Enabling Full Display Resolution

After installing VMware Tools, you might still be limited to a low resolution. To fix this, open Terminal in macOS and run: sudo /Library/Application\ Support/VMware\ Tools/vmware-resolutionSet 1920 1080 (replace with your desired resolution).

Improving Performance

In your .vmx file, add MemTrimRate = "0" to ensure the VM has exclusive access to its allocated RAM. Store VM files on an SSD for the best disk I/O performance.

Troubleshooting

Stuck on Apple Logo (Boot Loop)?

This is almost always a typo in the .vmx file. Meticulously check every line you added. Ensure your board-id and hw.model are a matching pair from a real Mac.

For AMD CPUs: Kernel Panics

Technical Reason

The CPUID mask tricks macOS into thinking it's running on an Intel CPU, allowing it to load the correct kernel extensions.

Solution: You must add a CPUID mask to your .vmx file.

cpuid.0.eax = "0000:..."
# (Full CPUID mask code)
